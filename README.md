# Driver Benefit Icon Skill

**Driver Benefit Icon Skill** 是一套用于司机端利益点与牵引场景图标绘制的本地 Skill。

它的目标不是单次生成一张图，而是帮助团队长期保持图标风格统一、语义明确、小尺寸可读。

---

## 适用场景

适用于司机端上的：

- 利益点感知
- 任务牵引
- 奖励提醒
- 权益入口
- 业务宣导
- 订单标签
- 服务状态提示
- 活动卡片 / banner / 弹层图标

---

## 推荐使用方式

### 第一步：填写业务 brief

```text
业务场景：
业务语义：
主物件：
辅助物件：
语义强化物件：
主色方向：
使用位置：
输出要求：
必须保留：
必须避免：
```

### 第二步：调用 skill

对 AI 说：

```text
调用 Driver Benefit Icon Skill，基于以下 brief 生成图标：
{填写 brief}
```

### 第三步：局部迭代

不要重新描述全部风格，尽量只说清楚局部改动：

```text
基于上一版继续调整：
保持 Driver Benefit Icon Skill 风格不变，只把 xxx 调整为 xxx。
不要增加元素，不要改变构图，不要加文字和 logo。
```

## 快速示例

```text
调用 Driver Benefit Icon Skill：

业务场景：冰鲜专送服务宣导
业务语义：冷链、保温、冰鲜商品、配送专业
主物件：打开的蓝色保温箱
辅助物件：龙虾
语义强化物件：冰块、1 瓶无品牌啤酒
主色方向：蓝色为主，橙红点缀
使用位置：司机端订单标签、配送中提示、服务宣导卡片
输出要求：透明背景 PNG，图标主体居中，小尺寸清晰
必须保留：轻 3D、圆润、软高光、司机端利益触点图标风格
必须避免：文字、品牌 logo、真实背景、复杂纹理、过多元素
```

## 质量检查

图标生成后，请检查：

1. 缩小到 48px 是否还能看懂？
2. 是否 1 秒内能判断业务语义？
3. 是否只有 1 个主物件 + 1 到 2 个辅助物件？
4. 是否没有文字、logo、水印？
5. 是否和已有图标高光、圆角、体积感一致？
6. 是否适合放在红橙渐变卡片上？
7. 是否没有过度真实、过度复杂、过度插画化？
8. 是否可以直接导出透明 PNG 使用？
